## See It In Action: Code & Output

Analytics should be auditable, readable code. The LunivCore DSL lets engineers write reports with clean, minimal syntax.

<Tabs starlight>
    <TabItem label="LCore DSL">
        Write analytics like code:

    ```bash
    dataset Sales "Q1 Revenue by Region" {
        APAC: 2850000
        Americas: 3450000
        EMEA: 1920000
    }

    plot Sales as bar
    ```

    #### Output (Generated by the C Core):

    ```text
    ┌─ Q1 Revenue by Region
    ├─────────────────────────────────────────────────────────────────
      APAC       │ ████████████████████████████████ (2850000)
      Americas   │ ███████████████████████████████████████ (3450000)
      EMEA       │ ██████████████████████ (1920000)
    └─────────────────────────────────────────────────────────────────
    ```
    </TabItem>
    <TabItem label="Lua Integration">
   Build complex, reusable logic:

    ```lua
    local sales = BI.new("YoY Growth Analysis")

    local current_revenue = 8700000
    local last_year_revenue = 6840000
    local growth_rate = ((current_revenue - last_year_revenue) / last_year_revenue) * 100

    if growth_rate > 20 then
        sales:add_kpi("Growth Rate (%)", growth_rate, 1) -- Value, Previous Value
        sales:add("Status", "High Growth")
    end

    sales:plot_kpi()
    ```
    </TabItem>
</Tabs>

## Architecture: C Performance, Lua Flexibility

LunivCore leverages the best of two worlds to ensure speed and extensibility. 

<CardGrid stagger>
    <Card title="Host Language (C)" icon="rocket">
        The C core efficiently **manages memory** and executes **computationally intensive tasks** (Lexer, Parser, Executor) for high-speed data processing.
    </Card>
    <Card title="Guest Language (Lua)" icon="pencil">
        Provides a **clean, dynamic scripting interface (DSL)** for users, which handles complex logic and communicates with the C core.
    </Card>
    <Card title="The Glue: Lua C API" icon="right-arrow">
        This stable API acts as the secure, high-speed bridge, efficiently **passing data and commands** between the C and Lua environments.
    </Card>
</CardGrid>
